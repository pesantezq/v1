# Claude Code — Health Remediation Prompt (Type A)

Type A prompts (something is broken/degraded — e.g. "FMP budget artifact stale,
inspect/fix/test/report") are generated by the **operator-control plane**, not by
this next-stage layer:

- Generator: `operator_control/worker_prompts.py` (`generate_prompt`)
- Created via: `POST /dashboard/operator/create` → work order → prompt
- Runner: `operator_control/worker_runner.py` (scaffold default; gated autonomous)

This file exists so all three prompt types are documented together. Type A prompts
carry the same forbidden block as Type B/C (no auto-trading / order placement /
broker writes / money movement / protected-logic edits / unrelated refactors) plus
the operator-control protected-path quarantine guard. See `docs/operator_control.md`
and `docs/operator_control_worker_runner.md`.
