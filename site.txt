
A. Root causes (confirmed)

theme_engine/theme_store.py was overwriting outputs/latest/theme_signals.json with an empty themes list when RSS produced no new items, so a repeated run could erase the last valid theme snapshot.
watchlist_scanner/system_summary.py only trusted current latest theme artifacts plus the previous summary. If latest theme artifacts were empty, top_theme could become empty even when a prior valid theme still existed.
watchlist_scanner/scanner.py was running fallback before cooldown, so cooldown could remove all alerts afterward and leave nothing.
watchlist_scanner/postprocess.py was also dropping post-cooldown fallback opportunities from final alerts, so even when fallback was added later in the flow it could disappear from the persisted output.
watchlist_scanner/__main__.py dry-run output writing needed to persist the refreshed watchlist artifact so the requested validation command checked the current run instead of stale files.

C. New logic flow (before vs after)

Before:

Theme run with no new RSS wrote empty latest theme artifacts.
system_summary could emit top_theme = {}.
Scanner fallback ran before cooldown.
Cooldown could remove all alerts after fallback selection.
Postprocess could suppress/drop fallback opportunities from final alerts.
After:

If a theme run produces no themes, ThemeStore.save_signals() preserves the prior latest theme snapshot and marks it no_update=True with theme_source="stale".
system_summary resolves top_theme in this order:
current non-empty theme data
most recent non-empty historical theme snapshot
previous system_decision_summary.json
top_theme now carries theme_source: "fresh" or "stale".
Scanner no longer injects fallback before cooldown.
Final flow is now:
alerts after filters
alerts after cooldown
if none remain, run fallback top-N using the existing minimum signal threshold
scan_summary now includes:
fallback_alerts_used
fallback_alert_count
fallback_trigger_stage = "post_cooldown"
Postprocess preserves fallback opportunities in final outputs as advisory opportunities rather than dropping them.
